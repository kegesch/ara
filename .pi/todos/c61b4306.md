## Problem
Adding a new entity type requires touching 21 files with 141 string references. Entity-type knowledge is spread horizontally across every layer via `switch` statements in 16 locations.

## Solution
Create a descriptor interface that encapsulates all type-specific behaviour behind a single object per entity type:

```typescript
export interface EntityDescriptor<T extends Entity> {
  type: T['type'];
  prefix: string;
  folder: string;
  statuses: string[];
  defaultStatus: string;
  color: (id: string) => string;
  icon: (status: string) => string;
  edges: (entity: T) => Array<{ to: string; type: EdgeType }>;
  parse: (meta: RawFrontmatter, base: EntityBase) => T;
  serialize: (entity: T) => string[];
  template: (title: string) => string;
  // extend as needed
}
```

## Target structure
```
src/entities/
  requirement/index.ts
  assumption/index.ts
  decision/index.ts
  idea/index.ts
  registry.ts          → Map<EntityType, EntityDescriptor>
```

## Benefit
Adding entity type #5 becomes: create one new directory, implement one descriptor, register it. No switch statements to hunt down.
