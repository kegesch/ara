## Problem
When an assumption is promoted to a requirement (`arad promote A-001`), a new requirement (e.g. R-016) is created. But the decisions that were `driven_by` the assumption still only reference the old assumption ID. They don't get linked to the new requirement.

## Example
- A-001 (validated) → promoted to R-016
- D-001 has `driven_by: [A-001]`
- D-001 does NOT get `driven_by: [A-001, R-016]`
- Result: R-016 is an orphan requirement

## Fix
In `src/commands/validate.ts` `promoteCommand()`, after creating the new requirement, find all decisions that reference the assumption in their `driven_by` and add the new requirement ID as well. Or at minimum, add a note/warning that the user should re-link.

## Where
`src/commands/validate.ts` — the `promoteCommand` function
