## Goal
Add a command to create relationships between entities that already exist, without editing the markdown files manually.

## Scope
- Add `arad link <from-id> <to-id> [--type <edge-type>]` command
- Supported edge types:
  - `driven_by` — link a decision to a requirement or assumption that drives it
  - `enables` — link a decision to another decision it enables
  - `derived_from` — link a requirement to its parent requirement
  - `conflicts_with` — mark two requirements as conflicting
  - `supersedes` — mark a decision as superseding an older one
- Validate the edge type is valid for the entity types involved (e.g., only decisions can have `driven_by`)
- Prevent self-referential links and duplicate links
- Update the frontmatter on disk directly
- Add `arad unlink <from-id> <to-id>` to remove a relationship

## Why
Currently, relationships are only set at `arad add` time. Real usage is iterative — you add entities first, then discover and link relationships later.
