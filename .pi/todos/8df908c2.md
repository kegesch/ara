## Problem
There is no `arad remove` or `arad delete` command. Once an entity is created, you can only `edit` it or manually delete the file. There's no way to cleanly remove an entity and warn about dangling references.

## What should happen
`arad remove D-001` should:
1. Check if other entities reference D-001 (dependents)
2. Warn the user: "D-001 is referenced by D-002 (enables). Remove anyway? [y/N]"
3. Remove the file
4. Optionally clean up references from other entities' frontmatter

## Current workaround
Delete the file manually, then run `arad check` to find the dangling references.

## Where
New file: `src/commands/remove.ts`
