## Completed

Refactored all commands to follow the `check.ts` pattern of pure functions + CLI wrappers:

### New files
- `src/core/errors.ts` — 15 typed error classes for structured error handling

### Refactored commands
Each command now exports pure functions that return structured data:
- `add.ts` → `createEntity(dir, input): CreateEntityResult`
- `link.ts` → `performLink(dir, fromId, toId, opts): LinkResult` + `performUnlink(...): UnlinkResult`
- `remove.ts` → `performRemove(dir, id, opts): RemoveResult`
- `rename.ts` → `performRename(dir, id, newId, opts): RenameResult`
- `validate.ts` → `performValidate()`, `performInvalidate()`, `performPromote()`
- `status.ts` → `getStatus(dir): StatusResult`

### MCP server
- Shrunk from 744 → 464 lines (38% reduction)
- Now delegates to pure functions instead of re-implementing logic
- Added 3 new MCP tools that were previously missing: `arad_remove`, `arad_rename`, `arad_status`
- Centralized error handling via `withErrorHandler()` wrapper
- Added `toolResult()` / `toolError()` helpers for consistent response formatting

### Results
- 91/91 tests pass
- TypeScript compiles cleanly (`tsc --noEmit` — 0 errors, previously had type errors)
- Binary builds successfully
