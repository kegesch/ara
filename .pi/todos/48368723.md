## Problem
Most commands are coupled to `console.log` and `process.exit`. This forces the MCP server (790 lines, ~20% of codebase) to re-implement every command's logic inline. Adding a new surface (REST API, lib import, etc.) would require yet another re-implementation.

## Solution
Separate pure logic from I/O in every command. Follow the pattern already used by `check.ts`:

```typescript
// check.ts already does this right:
export function runCheck(): CheckResult { ... }  // pure logic, returns structured data
export function checkCommand(): void {            // thin CLI wrapper, calls runCheck()
  const result = runCheck();
  displayText(result);
  process.exit(result.issues.length > 0 ? 1 : 0);
}
```

Apply this pattern to all commands:
- `add` → `createEntity()` returns the entity + path, `addCommand()` handles prompts + output
- `link` → `performLink()` returns the result, `linkCommand()` handles validation + output
- `rename` → `performRename()` returns old/new IDs, `renameCommand()` handles output
- `remove` → `performRemove()` returns removed entity + cleaned refs, `removeCommand()` handles output
- `validate` / `invalidate` / `promote` → already partially there, just need cleaner separation
- `status` → `getStatus()` returns structured summary, `statusCommand()` renders it

## Benefit
- MCP server shrinks from ~790 lines to ~100 lines (thin registration, delegates to pure functions)
- Any new surface (REST, SDK, test) calls the same pure functions
- Commands become testable without mocking `console.log` or `process.exit`
