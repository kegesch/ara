## Glossary / Ubiquitous Language

Add a glossary for shared term definitions.

### What
- New entity type: `term` with prefix `T`, folder `terms/`
- Statuses: `draft`, `accepted`, `deprecated`
- Template: definition, disambiguation, examples
- `arad add term "Order" --definition "A customer's intent to purchase one or more items"`
- Terms can be linked: `disambiguates_from` edge
- `arad check` warns about requirements/decisions using undefined terms (stretch goal)

### Why
Many contradictions stem from different meanings for the same word. A shared glossary prevents that before it becomes a conflict. Low overhead, high communication value.

- `src/types.ts` — `Term` interface, `ENTITY_CONFIG` entry
- `src/graph/graph.ts` — handle term edges
