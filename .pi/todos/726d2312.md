## Context

`arad check` reports issues accurately but doesn't suggest how to fix them. Example:

```
✗ [status_anomaly] accepted decision backed by invalidated A-011
```

The user has to figure out themselves what to do.

## Proposal

Add a `💡 Suggestion:` line after each issue:

- `status_anomaly` (invalidated assumption) → "Consider adding a requirement for the opposite and linking D-034 to it: `arad add requirement '...'` then `arad link D-034 <new-id>`"
- `orphan_requirement` → "No decision addresses this. Add a decision: `arad add decision '...'` then `arad link <new-id> R-016`"
- `dangling_ref` → "Remove the broken reference: `arad unlink D-005 R-999`"

Keep suggestions short and command-focused so users can copy-paste.
