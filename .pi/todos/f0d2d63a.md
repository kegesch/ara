## Problem
When multiple `arad add` commands run in parallel (e.g. from parallel bash calls), they all read the same highest ID from disk and compute the same next ID, causing duplicate IDs (R-020 was assigned to 6 entities simultaneously).

## Evidence
A-011 ("Parallel invocations won't happen") was **invalidated** during dogfooding — we saw 6 parallel `arad add requirement` calls all produce R-020.

## Fix ideas
- File-based locking (e.g. `.arad/.lock`)
- Atomic ID reservation via a counter file
- Pre-check for existing ID before writing (detect collision, retry)
