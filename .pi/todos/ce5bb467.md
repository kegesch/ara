## Problem
There's no `arad import` command to bring existing ADR (Architecture Decision Records) markdown files into ARAD format. Many teams already have ADRs in `docs/adr/` with a similar but different format (usually just markdown without the typed YAML frontmatter ARAD expects).

## What should happen
`arad import docs/adr/ --type=adr` should:
1. Scan the directory for markdown files
2. Parse each as an ADR (usually has title, status, context, decision, consequences sections)
3. Convert to ARAD decision entities with proper frontmatter
4. Assign sequential D-xxx IDs
5. Write to `.arad/decisions/`

## Where
New file: `src/commands/import.ts`
