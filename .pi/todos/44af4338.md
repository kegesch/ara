## Problem
There are no tests for the MCP server tools. The CLI commands are tested indirectly through the graph/search/parser unit tests, but the MCP layer that wraps them is untested. If the MCP tool signatures drift from the CLI behavior, we'd never know.

## What should happen
- Unit tests for each MCP tool (arad_list, arad_show, arad_trace, arad_impact, arad_check, arad_query, arad_add, arad_validate, arad_invalidate, arad_promote)
- Test that each tool returns valid JSON with the expected structure
- Test error cases (missing entity, not an ARAD project, wrong type)

## Where
`tests/mcp.test.ts`
