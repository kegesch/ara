## Problem\nThe `add` command accepts `--supersedes D-xxx` and writes the field to the new entity's frontmatter, but does **not** update the old decision's status to `superseded`.\n\nThe `link` command handles this correctly — when you link with `--type supersedes`, it flips the target decision's status automatically.\n\n## Fix\nIn `src/commands/add.ts`, after the entity is written, if `meta.supersedes` is set and the superseded entity is a decision, read it, set status to `superseded`, and call `updateEntity`.\n\n## Repro\n```\narad add decision \"New decision\" --supersedes D-001\narad show D-001  # status is still \"accepted\", should be \"superseded\"\n```
