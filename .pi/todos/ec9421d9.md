## Fix implemented

In `src/commands/add.ts`, after `createEntity` writes the new entity, if `input.supersedes` is set and the referenced entity is a decision, it now:
1. Reads the old decision from disk
2. Sets its status to `superseded`
3. Calls `updateEntity` to persist

This matches the behavior already present in the `link` command.

### Tests added (in `tests/concurrency.test.ts`):
- Auto-marks old decision as superseded
- No-op if old decision already superseded
- No crash if supersedes ID doesn't exist (dangling ref)
